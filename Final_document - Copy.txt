Abstract
The Eversales system is a web-based platform designed to support both retail and wholesale buying and selling within a single system. The application allows buyers to browse products, add items to a cart, place orders, track delivery status, and generate invoices. Sellers can register as retailers or wholesalers and manage products with flexible pricing based on quantity, enabling bulk purchasing at discounted rates. The system also supports promotional features such as discounts, promo codes, and buy-one-get-one offers to improve customer engagement and sales.
Additionally, the application includes an admin module that manages seller approval, monitoring, and blocking to maintain platform reliability and security. Buyers can provide product ratings and reviews, helping improve trust and product quality transparency. The project aims to bridge the gap between traditional retail e-commerce and wholesale markets by providing a centralized, user-friendly, and efficient digital marketplace for buyers and sellers.

Chapter 1: Introduction
In the modern digital era, online shopping systems have become an essential part of daily life. People prefer online shopping because it provides convenience, saves time, and allows customers to purchase products from anywhere through the internet. E-commerce websites help businesses reach a larger number of customers and manage their products and services efficiently. The growing use of web technologies has increased the demand for secure and user-friendly online shopping platforms.
The project �Eversales� is an online shopping web application developed using PHP and RDBMS (MySQL). The system is designed to provide customers with an easy platform to browse products, create accounts, add items to a shopping cart, and place orders online at retail and wholesale rates. It also helps administrators manage products, customer information, and orders effectively through a centralized database system. This project provides practical knowledge of web development, database management, and dynamic website creation using PHP and MySQL.
1.1 Overview
�Eversales� is an online shopping web application developed using PHP and Relational Database Management System (RDBMS). The system is designed to provide users with a convenient platform for purchasing products online. The website allows customers to browse products, create accounts, add products to a shopping cart, and place orders easily through the internet.
The project aims to digitalize the traditional shopping system by providing a secure, fast, and user-friendly online shopping experience. The system also helps administrators manage products, customers, and orders efficiently through a centralized database system. Technologies such as HTML, CSS, JavaScript, PHP, and MySQL are used to develop the application.
1.2 Problem Statement
Traditional shopping systems require customers to visit physical stores to purchase products, which can be time-consuming and inconvenient. Many small businesses also face difficulties in managing product records, customer details, and sales manually.
Some common problems in traditional shopping systems are:
* Time-consuming shopping process 
* Limited shopping hours 
* Difficulty in managing customer and product records manually 
* Lack of proper data organization 
* Increased chances of human errors in billing and record keeping 
* To solve these problems, the �Eversales� online shopping website has been developed. The system provides an easy and efficient way for users to shop online and for administrators to manage products and customer data digitally.

1.5  Scope and Limitations
The Eversales System is a web-based ecommerce platform developed to support both retail and wholesale business operations within a single system. The project focuses on simplifying the process of product purchasing, order management, and user management for different types of users such as retail buyers, wholesale buyers, sellers, and administrators. The system allows sellers to manage products, stock, and quantity-based pricing, while buyers can browse products, add items to cart, place orders, and cancel orders when required. A major scope of the project is the implementation of MOQ (Minimum Order Quantity) order management, which enables wholesale buyers to purchase products in bulk at discounted prices. The system also provides authentication, role-based access control, order processing, and database management to ensure smooth communication between users and the platform. Overall, the project aims to create an efficient and centralized ecommerce solution suitable for both small-scale retail transactions and large-scale wholesale purchasing activities.
Limitations of the Ever Sales System
* The system may not support advanced online payment gateway integration. 
* Real-time shipment and delivery tracking features are not available. 
* The platform requires a stable internet connection to function properly. 
* Advanced security mechanisms such as two-factor authentication are not implemented. 
* Performance may decrease when handling a very large number of users or transactions. 
* The system does not provide AI-based product recommendations or analytics. 
* Inventory management features are limited to basic stock handling. 
* Mobile application support may not be available. 
* MOQ pricing and quantity settings require manual configuration by the seller. 
* Multi-language and multi-currency support are not included in the current system.

Chapter 3: Methodology 
The Ever Sales System will be developed as a web-based online shopping and wholesale platform serving four distinct user roles: retail buyer, wholesale buyer, seller, and admin. The system will be built using a modern web technology stack comprising React.js for the frontend, Node.js with Express.js for the backend server, and MySQL as the relational database management system. By leveraging JavaScript as a unified language across both the client and server layers, the development team can maintain consistency and efficiency throughout the codebase. The application will implement RESTful API architecture to handle communication between the frontend and backend, with JSON Web Tokens (JWT) used for secure session management and role-based access control across all user types.
3.1 Software Development Life Cycle
The Ever Sales System is developed following the Agile Software Development Life Cycle (SDLC). This methodology was selected because it supports iterative development, allowing the team to deliver functional modules incrementally and refine each one based on testing feedback before proceeding to the next. Given the complexity of the system - which must simultaneously support retail purchasing, wholesale bulk ordering with MOQ validation, seller product and order management, and a comprehensive admin control panel - the Agile approach ensures that each role's features are thoroughly developed and tested before integration. The four primary phases of the SDLC applied to this project are as follows:
Requirements: Through comprehensive analysis of the system context diagram and
DFD Level 1, combined with review of existing e-commerce and wholesale platforms, the key functional requirements of the Ever Sales System were identified. These include user authentication with role-based access for retail buyers, wholesale buyers, sellers, and admins; product browsing and search; add to cart; promo code application; retail and wholesale order placement with cancellation; MOQ validation and wholesale pricing; seller product management including pricing, inventory, and variants; seller order management covering dispatch, status tracking, and order history: seller-buyer.
3.2 System Design
Detailed system architecture was designed based on the confirmed requirements. This includes the database scheme covering all twelve data stores identified in the DFD Level 1, the MVC (Model-View-Controller) architectural pattern to ensure separation of concerns and maintainability, user interface wireframes for all four role-specific dashboards, and the full DFD documentation at both Level 0 and Level 1 using Gane-Sarson notation. A project timeline with clear milestones was established to guide the development process across the fourteen-week schedule.

Chapter 5: System Design 
5.1 System Architecture
The Ever Sales System follows a three-tier web application architecture consisting of the Presentation Layer, Application Layer, and Database Layer. The system supports retail buyers, wholesale buyers, sellers, and administrators through a centralized ecommerce platform.
The Presentation Layer provides the user interface where users can log in, browse products, manage carts, place orders, and manage products. The Application Layer handles the business logic such as authentication, order processing, MOQ validation, quantity-based pricing, and product management. The Database Layer stores data related to users, products, carts, retail orders, and wholesale MOQ orders.
The architecture supports role-based access control, allowing each user type to access specific functionalities. This structure ensures smooth communication, efficient data management, and organized retail and wholesale ecommerce operations

6.1 Programming Platform
In this project, we used a programming platform consisting of the software development environment and tools required to design, develop, and test the application efficiently. The main components of the programming platform are:
Programming Language
The Eversales project is developed using PHP along with HTML, CSS, JavaScript, and MySQL. PHP is a widely used server-side scripting language that is suitable for developing dynamic web applications. HTML and CSS are used for designing the structure and layout of web pages, while JavaScript is used to add interactivity and improve user experience.
Integrated Development Environment (IDE)
The main IDE used for developing, debugging, and managing the project is Visual Studio Code (VS Code). VS Code is lightweight, fast, and supports various extensions useful for web development such as PHP extensions, Live Server, and MySQL tools. Its integrated terminal, intelligent code suggestions, and debugging features help in efficient development and project management.
Database Management System
MySQL is used as the relational database management system for storing and managing data related to users, products, orders, and transactions. It provides efficient data handling, security, and easy integration with PHP.
Libraries and Frameworks
The project uses Bootstrap for responsive user interface design and JavaScript libraries for enhancing website interactivity. PHP libraries and MySQL functions are used to manage database connectivity and perform CRUD (Create, Read, Update, Delete) operations efficiently.
